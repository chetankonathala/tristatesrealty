All motion components MUST be "use client" components. All MUST respect `prefers-reduced-motion` by checking the media query and disabling animations when reduced motion is preferred.

1. **FadeIn** — `src/components/motion/fade-in.tsx`:
Per UI-SPEC scroll reveal spec:
- Initial: `opacity: 0, y: 24`
- Animate: `opacity: 1, y: 0`
- Duration: 500ms, ease: "easeOut"
- Viewport: `{ once: true, amount: 0.2 }`
- Props: `delay?: number`, `className?: string`, `children`
- Use `motion.div` with `whileInView`
- Add `style={{ willChange: "transform, opacity" }}`

Reduced motion: If `prefers-reduced-motion: reduce`, set initial = animate (no animation).

2. **StaggerChildren** — `src/components/motion/stagger-children.tsx`:
Container that staggers its children's entrance animations:
- Parent: `motion.div` with `whileInView` variants
- `staggerChildren: 0.1` (100ms per UI-SPEC, but research says 60ms — use 100ms as UI-SPEC is LOCKED)
- Children: each child wrapped in `motion.div` with fade + y animation
- Viewport: `{ once: true, amount: 0.15 }`
- Props: `className?: string`, `staggerDelay?: number` (default 0.1), `children`

Note: The children should be wrapped in motion.div individually. Use `React.Children.map` to wrap each child, OR require consumers to use `motion.div` children manually. The simpler approach: provide the parent with stagger variants and let FadeIn children handle their own animation.

Recommended approach: StaggerChildren is just the stagger container that provides the timing context. Children use FadeIn individually. The stagger effect comes from the parent's `transition.staggerChildren`.

3. **PageTransition** — `src/components/motion/page-transition.tsx`:
Per UI-SPEC page transition spec:
- Wrap children in `AnimatePresence mode="wait"`
- Key by `usePathname()`
- Initial: `opacity: 0, y: 8`
- Animate: `opacity: 1, y: 0`
- Exit: `opacity: 0, y: 8`
- Duration: 300ms, ease: "easeOut"

4. **ScrollReveal** — `src/components/motion/scroll-reveal.tsx`:
A simpler alternative to FadeIn for single elements:
- Same as FadeIn but with configurable direction (up, down, left, right)
- Default: fade + translate-y 24px (up)
- Props: `direction?: "up" | "down" | "left" | "right"`, `delay?: number`, `duration?: number`

5. Create barrel export `src/components/motion/index.ts`:
```typescript
export { FadeIn } from "./fade-in";
export { StaggerChildren } from "./stagger-children";
export { PageTransition } from "./page-transition";
export { ScrollReveal } from "./scroll-reveal";
```

CRITICAL: Only animate `opacity` and `transform` properties. Never animate `height`, `width`, `top`, `left` — these cause layout shift and fail DS-05 CLS requirements.
CRITICAL: All motion components must have `will-change: transform, opacity` for GPU compositing.
