## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (e.g., X icon inside location tag), inline padding |
| sm | 8px | Compact element spacing (gap between filter pills, chip gaps) |
| md | 16px | Default element spacing (popover internal padding, card grid gap) |
| lg | 24px | Section padding (card grid outer padding, filter body section spacing) |
| xl | 32px | Layout gaps (pagination vertical padding) |
| 2xl | 48px | Major section breaks (empty state vertical padding) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: Filter pill height fixed at 36px (h-9) per existing FilterPill component. Popover content width fixed at 320px (w-80) for Price/Type popovers and 240px (w-60) for Beds/Baths popovers.

## Interaction Contracts

### I-01: Desktop Filter Pill Popovers (D-05)

**Trigger:** Click on any of the 4 desktop FilterPills (Price, Beds, Baths, Home Type).
**Behavior:** Opens a Popover anchored below the pill, aligned to start (left edge). Only one popover open at a time -- opening one closes any other open popover. Click outside closes the popover (Radix default behavior).
**State:** Local `openPill: string | null` state in SearchFilters. `null` = all closed. Values: `"price"`, `"beds"`, `"baths"`, `"type"`.
**Content per pill:**

| Pill | PopoverContent Width | Component Inside |
|------|---------------------|-----------------|
| Price | w-80 (320px) | PriceRangeSlider |
| Beds | w-60 (240px) | BedsBathsSelector (label="Beds") |
| Baths | w-60 (240px) | BedsBathsSelector (label="Baths") |
| Home Type | w-80 (320px) | PropertyTypeChips |

**PopoverContent styling:** `align="start"` `sideOffset={8}`. Background: `--popover`. Border: `--border`. Border radius: `--radius-lg`.
**Focus trap note:** If arrow keys conflict between Popover focus trap and PriceRangeSlider's Radix Slider, apply `onOpenAutoFocus={(e) => e.preventDefault()}` on PopoverContent.
**Mobile:** Popovers are NOT rendered on mobile (< lg breakpoint). Mobile continues using the single "Filters" Sheet.

### I-02: ZIP/City Auto-Detection (D-01)

**Trigger:** User types a value and presses Enter in LocationSearch input.
**Behavior:** `add()` tests trimmed input against `/^\d{5}$/`. If match, value routes to `postalCodes` URL param. If no match, value routes to `cities` URL param. No validation warning for non-Delaware inputs (D-02 dropped).
**Visual:** ZIP tags render identically to city tags: `bg-accent/15 text-accent text-xs px-3 py-1 rounded-full font-bold` with X remove button. ZIPs and cities render in a single `flex flex-wrap gap-2` container (ZIPs first, then cities, or interleaved by insertion order).

### I-03: Delaware Preset Chips (D-03)

**Trigger:** Component mounts inside More Filters modal/sheet Location section.
**Behavior:** 8 preset city chips rendered below the text input: Lewes, Rehoboth Beach, Wilmington, Dover, Newark, Middletown, Millsboro, Milton. Tapping a chip calls `onCitiesChange` to add that city. Chips for already-selected cities are hidden (not disabled, hidden).
**Styling:** `bg-accent-muted text-accent text-xs px-3 py-1.5 rounded-full font-medium cursor-pointer hover:bg-accent/20 transition-colors`. Rendered in `flex flex-wrap gap-2` container below the input, above the selected tags.
**Label:** No heading needed -- placement below the input with visual styling makes purpose self-evident.

### I-04: Sort -- Days on Market (D-07)

**Trigger:** User opens sort dropdown in SearchResultsHeader.
**Behavior:** 6th option "Days on Market" added at the bottom of the dropdown. Selecting it sets `sort=dom-asc` in URL, which sorts by `days_on_market ASC` (freshest listings first, NULLs sorted to end via `nullsFirst: false`).
**Dropdown order:**
1. Newest (date-desc) -- default
2. Price: High to Low (price-desc)
3. Price: Low to High (price-asc)
4. Most Bedrooms (beds-desc)
5. Largest Sqft (sqft-desc)
6. Days on Market (dom-asc)

### I-05: Pagination (D-09, D-10)

**Trigger:** Results grid renders with listings.
**Behavior:** SearchPagination appears below the card grid. Displays "Page X of Y" centered between Prev and Next buttons. Page count: `Math.max(1, Math.ceil(totalCount / 24))`. URL param: `?page=N` via nuqs `parseAsInteger.withDefault(1)`.
**Button states:**
- Page 1: Prev button disabled (`opacity-50 pointer-events-none`)
- Last page: Next button disabled
- Single page of results: Both buttons disabled, still shows "Page 1 of 1"
**Styling:** Buttons use `variant="outline" size="sm"`. Page counter uses `text-sm text-muted-foreground tabular-nums`. Container: `flex items-center justify-center gap-4 py-8`.
**Page reset on filter change:** When any filter in SearchFilters changes, `page` is set to `null` (resets to default 1). This prevents showing an empty grid when filters narrow results below the current page.
**Hidden when zero results:** Pagination does not render when `totalCount === 0` (EmptyState renders instead).

### I-06: Map Bounds Restoration (D-12)

**Trigger:** Page loads with `?bounds=swLng,swLat,neLng,neLat` in URL.
**Behavior:** SearchMap reads `bounds` URL param on mount. If present, `handleLoad` callback parses bounds and calls `mapRef.current.fitBounds()` with `padding: 20, duration: 0` (instant, no animation). If absent, map defaults to Delaware-wide viewport: center `[-75.5, 39.0]`, zoom ~9.
**No geolocation prompt:** Map never requests user location.
